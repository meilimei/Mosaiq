## 1. 背景

v0.7 之前 Detection Lab 只活在 `packages/sdk/bench/` 下，跑法是命令行（`pnpm bench:all`），输出 `bench/results/<timestamp>/raw.json` + `report.md`。Desktop app 的 "Detection Lab" 按钮是占位——只是用 persona 浏览器打开 pixelscan / browserscan 让用户人眼看。

v0.8 目标：**让 desktop renderer 在不依赖 bench/ 的前提下完整跑、存、看 Detection Lab**，把 bench/ 沉淀的 12 站规则和 scorer 算法变成 SDK 公共能力，desktop main process 直接 import。

### 1.1 v0.7 → v0.8 关键差异

| 维度 | v0.7（bench-only） | v0.8（SDK + desktop） |
|---|---|---|
| 入口 | CLI `pnpm bench:all` | Desktop UI + 可选 CLI |
| 受众 | Mosaiq 开发者验证反指纹 | 终端用户验证自己 persona |
| Persona | Ephemeral `baseline-bench-<ts>` | 用户自己的 persona |
| 存储 | `bench/results/<ts>/` 临时目录 | `~/.mosaiq/detection-runs/<personaId>/<runId>.json` |
| 复用粒度 | bench 内部 | `@mosaiq/sdk` 公共 API |
| 评分 | `bench/report.ts` analyze\* hits push | `computeScore(raw)` 纯函数 |
| 进度反馈 | console.log | `RunProgressEvent` 流（IPC push） |
| 历史 | 看文件夹 | UI trend chart + per-run detail |

## 4. Phase 8.3 — Runner（Persona → DetectionRunRaw）

### 4.1 目标

提供纯 SDK API：

```ts
export async function runDetection(
  persona: Persona,
  options?: RunDetectionOptions,
): Promise<RunDetectionResult>;
```

把现在散在 `@/d:/projects/Mosaiq/packages/sdk/bench/baseline-detection.ts` 里的 `runOne` / `runOneWithRetry` / persona launch / cleanup 流程，抽成可被 desktop main process 调用的纯 SDK 函数。

### 4.2 API 契约

```ts
// runner.ts
export interface RunDetectionOptions {
  /** 站点 id 子集；undefined = 全部 12 站。 */
  only?: readonly string[];
  /** 站点 id 黑名单。和 only 同时存在时 only 先生效然后 skip。 */
  skip?: readonly string[];
  /** 单站超时（默认 60_000ms）。 */
  timeoutMs?: number;
  /** 单站最大重试次数（默认 2，即首次失败后再重试 2 次共 3 attempts）。 */
  maxRetries?: number;
  /** 进度回调；按顺序触发 phase: 'site-start' | 'site-settle' | 'site-extract' | 'site-done' | 'site-error' | 'run-done'. */
  onProgress?: (evt: RunProgressEvent) => void;
  /** 中断信号（用户在 UI cancel）。 */
  signal?: AbortSignal;
  /** 透传给 launchPersona 的选项；默认 { headless: true }。 */
  launchOptions?: { headless?: boolean };
  /**
   * 把 raw screenshot / html 写到哪个目录。
   * undefined = 不存大 artifact，只把 metrics/extracted 留在内存 raw 里。
   * desktop 调用方应当传 `~/.mosaiq/detection-runs/<personaId>/<runId>/`。
   */
  artifactDir?: string;
}

export interface RunDetectionResult {
  /** 写盘的 source of truth；shape 同 bench raw.json。 */
  raw: DetectionRunRaw;
  /** 即时算好的 DetectionScore，避免调用方再 import scorer。 */
  score: DetectionScore;
}
```

### 4.3 Phase 8.3a — Pure shell + DI

把 runner 拆成可单测的纯壳：

```ts
// runner-core.ts (internal)
export interface SiteWorker {
  (spec: SiteSpec, ctx: SiteWorkerContext): Promise<SiteResult>;
}

export interface SiteWorkerContext {
  timeoutMs: number;
  artifactDir?: string;
  signal?: AbortSignal;
}

export async function executeRun(
  sites: readonly SiteSpec[],
  worker: SiteWorker,
  options: ExecuteRunOptions,
): Promise<DetectionRunRaw>;
```

`executeRun` 负责：
- 按 `only` / `skip` 过滤站点
- 顺序调 `worker(spec, ctx)`（已含 retry 逻辑用 `runWithRetry(worker, ...)`）
- 发 `RunProgressEvent`（site-start / site-done / run-done）
- 响应 `AbortSignal`（中断时把剩余站点标 `ok: false, error: 'aborted'`）
- 聚合 `SiteResult[]` 成 `DetectionRunRaw`

测试策略：mock `SiteWorker`，验证 retry 退避、progress 顺序、abort 行为、only/skip 过滤——全部不起 Playwright。

**预计**：runner-core.ts ~150 LOC + runner-core.test.ts ~250 LOC (~15 tests)；2-3h。

### 4.4 Phase 8.3b — Wire launchPersona

```ts
// runner.ts
export async function runDetection(persona, options): Promise<RunDetectionResult> {
  const sites = SITES;
  const session = await launchPersona(persona, options?.launchOptions ?? { headless: true });
  const page = await session.firstPage();

  const worker: SiteWorker = async (spec, ctx) => runOnePage(page, spec, ctx);

  try {
    const raw = await executeRun(sites, worker, {
      ...options,
      personaId: persona.metadata.id,
    });
    const score = computeScore(raw);
    return { raw, score };
  } finally {
    await session.close();
  }
}
```

`runOnePage` 把 bench/baseline-detection.ts 的 runOne 内逻辑（goto / settle / screenshot / extract）抽出来，**默认不写大 artifact**，仅当 `artifactDir` 传入时才落盘 screenshot/html。

测试策略：
- **不写** Playwright 集成测试（CI 起 chromium 太重；bench 已经覆盖这块）
- 写 e2e smoke：调 `runDetection(persona, { only: ['sannysoft'], headless: true })` 验证 raw.results 长度 + ok:true（手工跑，CI 跳过）
- typecheck 必过

**预计**：runner.ts ~120 LOC；2-3h（大部分时间在剪 baseline-detection.ts 抽出 runOnePage）。

## 5. Phase 8.4 — Storage

### 5.1 路径约定

跟随现有 `~/.mosaiq/personas/<id>.json` 风格：

```
~/.mosaiq/
  personas/
    <personaId>.json
  profiles/
    <personaId>/        # browser user-data-dir
  detection-runs/
    <personaId>/
      <runId>.json      # 完整 DetectionRun（含 raw + score）
      <runId>/          # artifacts（可选，仅当 8.3 传 artifactDir）
        sannysoft.html
        sannysoft.png
        ...
```

`runId` = `<ISO timestamp w/ '-' replace>` ；与 bench/results/ 时间戳同风格。

### 5.2 API

```ts
// run-store.ts
import type { DetectionRun, DetectionRunSummary } from './types.js';
import type { PathConfig } from '../paths.js';
import type { PersonaId } from '@mosaiq/persona-schema';

export interface DetectionRunSummary {
  runId: string;
  personaId: PersonaId;
  timestamp: string;
  durationMs: number;
  sitesAttempted: number;
  sitesOk: number;
  sitesFail: number;
  totalHits: number;        // = score.hits.length
  weightedHits: number;     // = score.weightedHits
}

export function saveDetectionRun(
  personaId: PersonaId,
  run: DetectionRun,

export function loadDetectionRun(
  personaId: PersonaId,
  runId: string,
  config?: PathConfig,
): DetectionRun;

export function listDetectionRuns(
  personaId: PersonaId,
  config?: PathConfig,
): DetectionRunSummary[];  // 仅元数据，避免加载全部 raw
```

`listDetectionRuns` 不读 raw 数据——直接读每个 .json 顶层字段构造 summary，避免 100 个历史 run 时一次性 OOM。

### 6.2 main.ts 实现要点

```ts
const activeRuns = new Map<string, AbortController>();

ipcMain.handle(IPC_CHANNELS.detectionLabRun, async (_evt, personaId) => {
  const persona = loadPersona(personaId);
  const runId = new Date().toISOString().replace(/[:.]/g, '-');
  const abort = new AbortController();
  activeRuns.set(runId, abort);

  // Fire-and-forget：异步跑，进度通过 webContents.send 推
  (async () => {
    try {
      const { raw, score } = await runDetection(persona, {
        signal: abort.signal,
        onProgress: (evt) => {
          mainWindow?.webContents.send(IPC_EVENTS.detectionLabProgress, { runId, progress: evt });
        },
        artifactDir: getDetectionRunArtifactDir(personaId, runId),
      });
      const run: DetectionRun = { runId, personaId, status: 'completed', raw, score };
      saveDetectionRun(personaId, run);
      mainWindow?.webContents.send(IPC_EVENTS.detectionLabProgress, {
        runId, progress: { phase: 'run-done', score },
      });
    } catch (err) {
      const failed: DetectionRun = { runId, personaId, status: 'failed', error: (err as Error).message };
      saveDetectionRun(personaId, failed);
      // 推 run-error event ...
    } finally {
      activeRuns.delete(runId);
    }
  })();

  return { ok: true as const, runId };
});

ipcMain.handle(IPC_CHANNELS.detectionLabCancel, async (_evt, runId: string) => {
  const ctl = activeRuns.get(runId);
  if (!ctl) return false;
  ctl.abort();
  return true;
});

// list / get / delete: 直接转发到 run-store helpers
```

### 6.3 重要：替换现有 `openDetectionLab`

`@/d:/projects/Mosaiq/apps/desktop/electron/main.ts:249-262` 当前实现是"在已启动 persona 浏览器上打开 pixelscan + browserscan 两站给用户人眼看"——这是占位行为，v0.8 之后**不再合理**（detection lab 是独立功能，与 persona launch 解耦）。

迁移路径：
1. **保留** `openDetectionLab` channel 名（向后兼容现有 `PersonaListPage.handleDetectionLab`）
2. **改实现** 为：检查 persona 是否存在 → 返回 `{ ok: true }`，让 renderer 跳转到新 `DetectionLabPage`
3. renderer 端的"在浏览器开 pixelscan"行为整体移除（被 `New Run` button 替代）

或者更干净：直接 deprecate `openDetectionLab` channel，renderer 不再调它而是直接 `setPage({ kind: 'detectionLab', personaId })`。**推荐这条**——零 round-trip 更快。

### 6.4 测试

主进程 IPC 现状没有 unit test 基础（main.ts 整文件无测试）。8.5 沿用约定不强求，但保 typecheck clean。

手工验证 checklist：
- [ ] 启动 detection run → 立刻拿到 runId

- [ ] cancel 期间进程不挂

## 8. 风险登记表

| # | 风险 | 影响 | 缓解 |
|---|---|---|---|
| R1 | 单 run 5-10 分钟（12 站 \* 平均 30-60s） | UX 难受，用户以为卡死 | 进度条 + per-site phase（goto / settle / extract）显式可见；cancel 按钮第一公民 |
| R2 | run 期间 app 退出/崩溃 → ephemeral persona / userDataDir 残留 | 磁盘垃圾累积 | persona-store 已有 `personaExists` 检查；启动时扫 `detection-runs/<id>/<runId>.json` 状态 `running` 的全标 `failed` |
| R3 | Detection run 与 persona 普通 launch 互斥 | 用户启动 persona 浏览时不能跑 run | 直接禁用 detection run（main.ts 检查 `runningSessions.has(personaId)`）；或用独立 ephemeral profile（推荐后者，但 8.3 先简单串行禁用） |
| R4 | recharts bundle size 增加 80kb gz | desktop dmg 略大 | 可接受；如果后续 size 敏感再换 d3 手搓 |
| R5 | scorer 算法未来演进会让历史 run 的 score 字段失效 | trend chart 出现"算法切换断点" | DetectionRun 存 raw（不可变），score 是派生；renderer 端可选择"用当前 scorer 重算所有历史"或显示 score schema version |
| R6 | bench retry 间 1-2-4s 指数退避在 UI 期间会看起来 stuck | 用户以为崩了 | progress event 在 retry 间也发 `phase: 'site-retry-backoff'` |

**Total: 14-24h，建议拆 4-6 个 session 推**。每个 phase 独立可 commit，不必等下一 phase 完成。

推荐顺序：8.3a → 8.3b → 8.4 → 8.5 → 8.6。理由：
- 8.3a 纯单测可独立验证；8.3b 之前 desktop UI 不能展示真数据
- 8.4 在 8.3b 之后做，因为 storage 的 IO contract 跟 runner 的 raw 落盘逻辑耦合
- 8.5 / 8.6 可以并行，但同一人推 8.5 → 8.6 顺序更省脑

## 10. Out of Scope（推到 v0.9+）

- **Detection Lab CI 集成**：把 detection run 加进 GitHub Actions 跑 chromium 太重，不在 v0.8。
- **Persona pool comparison**：跨多个 persona 同时跑 detection 做横向对比——v0.9。
- **Detection result export/share**：把一次 run 导出成 shareable URL / 报告 PDF——v0.9。
- **Detection Lab CLI**：单独的 `pnpm mosaiq detection-lab run <persona-id>` 命令——v0.9（8.3 完成后 bench/baseline-detection.ts 已经很接近这个，区别只是入参从临时 persona 改为指定 personaId）。
- **Persona auto-tune from detection results**：基于 detection score 自动调 noise amplitude / WebGL profile——v1.0+，需要 marketplace 数据。
- **Real-hardware detection capture pipeline**：让真机用户提交他们的 detection raw 做 baseline 校准——v0.9 + chromium-fork 后。
