## 1. 背景

v0.7 之前 Detection Lab 只活在 `packages/sdk/bench/` 下，跑法是命令行（`pnpm bench:all`），输出 `bench/results/<timestamp>/raw.json` + `report.md`。Desktop app 的 "Detection Lab" 按钮是占位——只是用 persona 浏览器打开 pixelscan / browserscan 让用户人眼看。

v0.8 目标：**让 desktop renderer 在不依赖 bench/ 的前提下完整跑、存、看 Detection Lab**，把 bench/ 沉淀的 12 站规则和 scorer 算法变成 SDK 公共能力，desktop main process 直接 import。

### 1.1 v0.7 → v0.8 关键差异

| 维度 | v0.7（bench-only） | v0.8（SDK + desktop） |
|---|---|---|
| 入口 | CLI `pnpm bench:all` | Desktop UI + 可选 CLI |
| 受众 | Mosaiq 开发者验证反指纹 | 终端用户验证自己 persona |
| Persona | Ephemeral `baseline-bench-<ts>` | 用户自己的 persona |
| 存储 | `bench/results/<ts>/` 临时目录 | `~/.mosaiq/detection-runs/<personaId>/<runId>.json` |
| 复用粒度 | bench 内部 | `@runova/sdk` 公共 API |
| 评分 | `bench/report.ts` analyze\* hits push | `computeScore(raw)` 纯函数 |
| 进度反馈 | console.log | `RunProgressEvent` 流（IPC push） |
| 历史 | 看文件夹 | UI trend chart + per-run detail |

把现在散在 `@/d:/projects/Mosaiq/packages/sdk/bench/baseline-detection.ts` 里的 `runOne` / `runOneWithRetry` / persona launch / cleanup 流程，抽成可被 desktop main process 调用的纯 SDK 函数。

### 4.2 API 契约

```ts
// runner.ts
export interface RunDetectionOptions {
  /** 站点 id 子集；undefined = 全部 12 站。 */
  only?: readonly string[];
  /** 站点 id 黑名单。和 only 同时存在时 only 先生效然后 skip。 */
  skip?: readonly string[];
  /** 单站超时（默认 60_000ms）。 */
  timeoutMs?: number;
  /** 单站最大重试次数（默认 2，即首次失败后再重试 2 次共 3 attempts）。 */
  maxRetries?: number;
  /** 进度回调；按顺序触发 phase: 'site-start' | 'site-settle' | 'site-extract' | 'site-done' | 'site-error' | 'run-done'. */
  onProgress?: (evt: RunProgressEvent) => void;
  /** 中断信号（用户在 UI cancel）。 */
  signal?: AbortSignal;
  /** 透传给 launchPersona 的选项；默认 { headless: true }。 */
  launchOptions?: { headless?: boolean };
  /**
   * 把 raw screenshot / html 写到哪个目录。
   * undefined = 不存大 artifact，只把 metrics/extracted 留在内存 raw 里。
   * desktop 调用方应当传 `~/.mosaiq/detection-runs/<personaId>/<runId>/`。
   */
  artifactDir?: string;
}

export interface RunDetectionResult {
  /** 写盘的 source of truth；shape 同 bench raw.json。 */
  raw: DetectionRunRaw;
  /** 即时算好的 DetectionScore，避免调用方再 import scorer。 */
  score: DetectionScore;
}
```

### 6.3 main.ts 关键决策

#### 单 persona 串行约束

```ts
const activeRuns = new Map<PersonaId, ActiveRunEntry>();           // 主索引：startRun 守卫
const activeRunsByRunId = new Map<string, ActiveRunEntry>();       // 反向索引：cancel 路径
```

键设计成 `PersonaId`（不是 `runId`）让 detectionLabRun 第二次调用 O(1) 命中冲突 → 返回 `ok:false`，避免两套 Playwright 抢同一个 user-data-dir。

#### Fire-and-forget + 终态由 main 接管

```ts
const FORWARD_PHASES: ReadonlySet<RunProgressEvent['phase']> = new Set([
  'init', 'site-start', 'site-end', 'site-retry',
]);

async function executeDetectionRunAsync(entry, persona) {
  try {
    const result = await runDetection(persona, {
      runId, signal: abort.signal,
      personaTemplate: getPersonaTemplate(persona),    // tag 反推 best-effort
      artifactDir: getDetectionRunArtifactDir(personaId, runId),
      onProgress: (evt) => {
        if (FORWARD_PHASES.has(evt.phase)) emitProgress(runId, evt);
        // ★ SDK 的 done/canceled/error 不透传——main 自己发带 finalRun 的终态
      },
    });
    const status = abort.signal.aborted ? 'canceled' : 'completed';
    const run = buildCompletedRun(runId, entry, result.raw, result.score, status);
    saveDetectionRun(personaId, run);
    emitProgress(runId, { runId, personaId, phase: ..., finalRun: run });
  } catch (err) {
    const run = buildFailedRun(runId, entry, err.message);
    try { saveDetectionRun(personaId, run); } catch (saveErr) { console.error(...); }
    emitProgress(runId, { runId, personaId, phase: 'error', error: err.message, finalRun: run });
  } finally {
    activeRuns.delete(personaId);
    activeRunsByRunId.delete(runId);
  }
}

ipcMain.handle(IPC_CHANNELS.detectionLabRun, (_evt, personaId): DetectionRunStartResult => {
  if (!personaExists(personaId)) return { ok: false, error: `Persona not found: ${personaId}` };
  if (activeRuns.has(personaId)) return { ok: false, error: 'already in-flight' };
  const persona = loadPersona(personaId);
  const entry = { runId: newRunId(), personaId, abort: new AbortController(), startedAt, startedAtMs };
  activeRuns.set(personaId, entry);
  activeRunsByRunId.set(entry.runId, entry);
  void executeDetectionRunAsync(entry, persona);   // fire-and-forget
  return { ok: true, runId: entry.runId };
});
```

理由：SDK 的 `done` 不知道 `DetectionRun.startedAt / finishedAt / meta`，也不应依赖 storage 决定持久化路径——这些是 main 的责任。SDK 终态被过滤、main 在 await 返回后构造 `DetectionRun` + `saveDetectionRun` + emit 唯一一个带 `finalRun` 的终态事件。

#### deleteDetectionRun 拒绝 in-flight

```ts
ipcMain.handle(IPC_CHANNELS.detectionLabDeleteRun, (_evt, personaId, runId): boolean => {
  if (activeRunsByRunId.has(runId)) {
    throw new Error(`Cannot delete in-flight run ${runId}; cancel it first then retry.`);
  }
  return deleteDetectionRun(personaId, runId);
});
```

防止 fire-and-forget 还在写 JSON / artifacts 时被删，造成半成品文件 + Windows EBUSY。

#### window-all-closed 不强求 await

```ts
for (const [, entry] of activeRuns) entry.abort.abort();
activeRuns.clear();
activeRunsByRunId.clear();
// 不 await fire-and-forget promise；强行 await 最坏 60s page.goto 卡住关窗
```

Trade-off：fire-and-forget 的 saveDetectionRun 可能来不及完成 → 下次 listDetectionRuns 看到坏 JSON warn skip（已被 §5 测试覆盖）。

### 6.4 落地差异 vs 设计稿

1. **`activeRuns` 键 = `PersonaId`** — 设计稿用 `runId`；改键后 startRun 守卫 O(1) 命中"单 persona 串行"约束，cancel 用 `activeRunsByRunId` 反向索引（O(1) lookup by runId）。
2. **SDK 终态过滤 + main 自发终态** — 设计稿 onProgress 直接转发；改成 main 拦截后由自己发带 `finalRun` 的唯一终态，避免 renderer 收到两条 `done`（一条 SDK 的无 finalRun + 一条 main 的有 finalRun）。
3. **`MosaiqEvents` 拆出独立 contextBridge expose** — 设计稿把 `onDetectionLabProgress` 塞进 `MosaiqApi`；改成单独 `window.mosaiqEvents.*`，与 invoke API 分开（语义清晰：API 是 request-response，Events 是 push-subscribe）。同时把 `onPersonaStopped` 也搬过去。
4. **`getPersonaTemplate(persona)` best-effort** — Persona schema 不持久化 template id；从 `metadata.tags` 里反查 known template id，找不到填 'unknown'。
5. **`SDK_VERSION` 显式 export 自 SDK** — 用 `src/version.ts` + 与 package.json 同步的 vitest 校验，不读 package.json 避免 dist 路径解析陷阱。
6. **保留 `openDetectionLab` channel** — 暂不动，让 8.6 renderer 改成跳转 `setPage({ kind: 'detectionLab' })` 时再 deprecate。

### 7.2 关键设计决策

#### `DetectionRun.raw` 内嵌（vs 设计稿的"按需 lazy load"）

设计稿担心 raw 序列化 30MB——但 `SiteResult.html` / `screenshot` 是相对**路径字符串**（实际 artifacts 在文件系统），所以 raw JSON 通常 <100KB。直接嵌入：

- detail page 一次 IPC `detectionLabGetRun` 拿到所有数据
- `listDetectionRuns` 投影 summary 时丢 raw（避免 100 个历史 run 一次性占内存）
- types.ts 头注 + DetectionRun JSDoc 显式标注"raw 可选 + summary 不带 raw"

#### 进度订阅 + 终态合并

```tsx
useEffect(() => {
  const cleanup = window.mosaiqEvents.onDetectionLabProgress((msg) => {
    if (activeRunIdRef.current !== msg.runId) return;  // ★ ref 过滤其它 run
    setActive((prev) => updateActive(prev, msg.progress));
    if (isTerminal(msg.progress.phase)) {
      activeRunIdRef.current = null;
      setActive(null);
      void refresh();                                  // 重拉历史列表
      toast(...phase 对应的语义...);
    }
  });
  return cleanup;
}, [refresh, toast]);
```

用 `useRef` 而不是 `useState` 存当前 runId：避免 closure 旧值导致事件被错误丢弃。

#### Surface 颜色 12 项 + severity 3 级

颜色映射 `SURFACE_BADGE_CLASS` 与 `SURFACE_COLOR_HEX` 双口径（CSS class 给徽标；HEX 给 recharts），保证雷达图轴顺序和列表 chip 视觉一致。

- webdriver / navigator → 红黄（high-priority）
- canvas / webgl → 紫蓝（最常 spoof）
- audio / font / webrtc / screen / permissions / timezone / plugins → 中性彩
- other → 灰

#### 单 persona 串行 UI 兜底

"New Run" 按钮在 `active != null` 时切换为"取消"按钮；列表里同 runId 的删除按钮 disabled。即使有人通过 console 强发第二次 startRun，main 也会返回 `ok:false` + toast.error。

#### Recharts 3.x 类型妥协

`<Tooltip>` 的 `labelFormatter` / `formatter` 在 recharts 3 里被 widen 成 `ReactNode | ValueType`。强制 narrow 用 `as SurfaceName | undefined` / `as number`，运行时安全（数据全是我们构造的）。

| # | 风险 | 影响 | 缓解 |
